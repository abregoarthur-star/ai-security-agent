PART 1 — POSITION IN FLEET
Tier separation mirrors enterprise orgs · tier 2 owns CVE/bounty, tier 3 owns AI-native
Brain (master hub — tier 1) │ 4 proxy tools: audit_mcp_server · diff_mcp_server │ firewall_recent_events · run_self_test │ └─▶ ai-security-agent /tools/* │ cron callback: /security/mcp-manifests ◀──┐ │ /security/firewall/events ◀─┤ ▼ │ Security Agent (tier 2 — traditional CVE/bounty) │ │ poll: /intel/ai-security (Phase 3 uplink) │ │ └─▶ ai-security-agent /intel/ai-security│ ▼ │ AI Security Agent (tier 3 — this service) ──────┘ │ ├── npm: @dj_abstract/mcp-audit (detect) ├── npm: @dj_abstract/agent-firewall (defend) ├── npm: @dj_abstract/agent-capability-inventory (inventory) └── npm: @dj_abstract/prompt-genesis (generate + selfTest orchestrator)

Why a separate tier?

All three tiers authenticate with a single shared AGENT_API_KEY, so tier-2 (CVE/bounty) and tier-3 (AI-native) can evolve independently without bundling the toolkit into their own dependency trees. Mirrors how enterprises run a dedicated AI security team alongside the traditional security team.

PART 3 — ENDPOINT REFERENCE
All /tools/*, /ai-security/*, and /intel/* routes gated on x-api-key when AGENT_API_KEY is set · /health and /architecture always public
Method Route Purpose Phase
GET /health Liveness probe. Reveals env-var-set booleans + tool list; no secrets. 1
GET /architecture This document. Public, no auth — same pattern as Brain's /architecture. 1
POST /tools/audit_mcp_server @dj_abstract/mcp-audit static scan of { manifest | manifestData | stdio | url }. 1
POST /tools/diff_mcp_server mcp-audit diff — rug-pull / drift between two reports. 1
POST /tools/firewall_recent_events Proxy to Brain's /security/firewall/events. Returns recent agent-firewall activity. 1
POST /tools/run_self_test Wraps @dj_abstract/prompt-genesis selfTest() — generate corpus, run through prompt-eval, return ship/hold decision. 2
POST /tools/capability_inventory @dj_abstract/agent-capability-inventory fleet tool catalog w/ sensitivity tags. Accepts inline manifestData or filesystem manifest path(s). 3
POST /tools/sweep_mcp_ecosystem Multi-target mcp-audit fan-out with bounded concurrency + fleet-level aggregation. Accepts targets[] or baselinesDir. 3
GET /ai-security/posture Raw cron-scan state + baseline count + last-scan timestamp. 1
GET /ai-security/last-scan Most recent scan summary (same shape as getLastScan()). 1
POST /ai-security/scan/trigger Manual scan kickoff — runs runSelfScan({ trigger: 'manual' }). 1
GET /intel/ai-security Consumer-ready snapshot: totals, per-server findings, high-priority findings, firewall 24h stats, freshness. Shaped for Security Agent polling. 3
Request/response shapes for each tool live next to the handler in src/tools/*.js and in the phase notes under .agents/.
PART 4 — CRON & STATE SHAPE
Managed by src/cron.js::startSchedule() · armed when Express starts listening
⏰ Daily Self-Scan
0 3 * * * · 03:00 UTC
Flow
Fetch Brain's manifests via GET /security/mcp-manifests
Audit brain-tools + brain-exec via mcp-audit
Diff against in-memory baseline (last good scan)
Critical drift → Telegram via Brain's POST /telegram/send
Daily Drift alerts
🚀 Boot One-Shot
+5s after app.listen()
Purpose
Establishes the baseline on cold boot
Proves the path end-to-end before cron fires
First post-restart scan re-seeds baseline (Phase B-1)
Startup In-memory only
State Shape — src/cron.js
{ baselines: Map<serverName, mcpAuditReport>, // in-memory, Phase B-1 lastRun: ISO8601 | null, lastScan: { trigger, startedAt, servers[], diffs[] } | null, lastError: { at, message } | null, }
The baseline is rebuilt on every scan (last good scan becomes the new baseline) so drift detection is always relative to the most recent clean state.

Railway secrets · solid border = required · dashed = optional

Required / Conditional

AGENT_API_KEY = shared fleet secret (x-api-key header)
BRAIN_API_URL = Brain production URL (cron dep)
BRAIN_API_KEY = Brain's API key for manifests + telegram
ANTHROPIC_API_KEY = prompt-eval judge (run_self_test)
GROQ_API_KEY = required if generator/target is Groq
API_KEY = legacy alias for AGENT_API_KEY
